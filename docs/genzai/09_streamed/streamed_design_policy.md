## 1. STREAMED公式ヘルプからの確認事実

### 1-1. 課税形式は関知しない（公式明記）

> STREAMEDでは税区分設定において課税形式の指定項目がありません。
> 免税事業者、簡易課税、原則課税（一括比例配分方式／個別対応方式）などの
> 課税形式を変更することで税区分の内容を切り替えたり
> データ化の際の税区分の判定方法を変更するということができません。
>
> — help/8090「課税形式について」

### 1-2. 税区分の自動判定（help/8369）

STREAMEDの自動判定対象は以下のみ:

| 自動判定種別 | 判定結果 |
|------------|---------|
| 自動判定(仕入) | 課税仕入10% / 課税仕入8% / 課税仕入(軽)8% / 非課税仕入 |
| 自動判定(売上) | 課税売上10% / 課税売上8% / 課税売上(軽)8% / 非課税売上 |
| 自動判定(売上貸倒) | 課税売上貸倒10% / 課税売上貸倒8% / 課税売上貸倒(軽)8% / 非課税売上 |

- 初期設定以外の税区分を自動判定の対象に追加**不可**
- 簡易課税の税区分は初期設定に**含まれない**（手動追加要）
- 「共通対応仕入」も初期設定に**含まれない**（手動追加要）

### 1-3. 税区分の設定項目（help/8090）

| 項目 | 説明 |
|------|------|
| 税区分名 | 会計ソフトと一致させる必要あり |
| 税区分コード | 会計ソフトがコード連携する場合のみ |
| 税率コード | 同上 |
| 適格判定対象 | チェックあり=適格判定を行う |

### 1-4. MF連携の注意点（help/8090）

- MFは税区分を**「名称」または「省略名」のどちらでもマッチング**する（2026-03-25 MF公式ヘルプ確認済み）
- MFは「適格判定情報を追加出力する」タイプの会計ソフト

### 1-5. STREAMEDが決めるもの・決めないもの

| STREAMEDが決める | STREAMEDが決めない |
|:---:|:---:|
| 勘定科目 | 税込/税抜の変換 |
| 税区分（10%/8%/軽8%/非課税） | 仮払消費税の生成 |
| 適格判定（T番号＋日付） | 課税形式（免税/簡易/原則） |
| 税込金額 | 経過措置の控除割合計算 |

## 2. 自社エンジンの責務定義

STREAMEDと同じ責務分離を採用する。

### エンジンの責務（するべきこと）

| # | 責務 | 説明 |
|---|------|------|
| 1 | 勘定科目を決める | MFの科目名に準拠 |
| 2 | 税区分を決める | MFの税区分「名称」に準拠 |
| 3 | 税込金額を保持する | 常に税込。税抜変換しない |
| 4 | 適格判定する | T番号＋日付で判定 |
| 5 | OCR情報を抽出する | 日付・金額・発行者等 |
| 6 | 控除明細(adjustments)を検出する | 源泉徴収・振込手数料等 |

### エンジンの責務外（MFに委譲）

| # | 責務外 | 処理する側 |
|---|--------|-----------|
| 1 | 税込経理/税抜経理の変換 | MF（ユーザー設定） |
| 2 | 仮払消費税の生成 | MF（税抜経理時に自動生成） |
| 3 | 仮受消費税の生成 | MF（売上側も同様） |
| 4 | 課税形式による集計 | MF（申告時） |
| 5 | 経過措置の控除割合 | MF（80%→70%等） |

## 3. v1テスト計画への影響

### 削除対象

| 項目 | 理由 |
|------|------|
| P-4課題（税込/税抜出力定義なし） | エンジンは常に税込。課題消滅 |
| T-2課題（P-4の実装手段） | 同上 |
| B9禁止科目チェック | TAX_PAID/TAX_RECEIVEDは科目リストから除外 |
| config動的反映（accounting_method切替） | 不要。常に税込 |
| test_config.tsの5パターン | 1パターンで十分 |

### 残すもの

| 項目 | 理由 |
|------|------|
| D-1課題（total_amount（合計金額）定義） | 契約対価（税込）= そのまま有効 |
| S-6課題（adjustments（控除明細）追加） | 控除明細は引き続き必要 |
| D-2課題（仕訳摘要ルール） | そのまま有効 |
| B10 adjustments（控除明細）整合チェック | そのまま有効 |
