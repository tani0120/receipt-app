## 4. 重大差異（要判断）

### 差異1: 備品・消耗品費 vs 消耗品費

| ソース | 科目名 |
|--------|--------|
| STREAMED | 備品・消耗品費 |
| MF CSV（個人） | 消耗品費 |
| 本システム旧enum | SUPPLIES（消耗品費） |

> STREAMEDでは2行存在（検索キーSHOUMOU/JIMUYOU）。
> MFインポート時にどちらの名称が使われるか要確認。
> MFの科目名が「消耗品費」なら、CSVには「消耗品費」を出力すべき。

### 差異2: 業務委託料 vs 外注工賃

| ソース | 科目名 |
|--------|--------|
| STREAMED | 業務委託料 |
| MF CSV（個人） | 外注工賃 |
| 本システム旧enum | OUTSOURCING（外注費） |

> MF法人は「外注費」、MF個人は「外注工賃」。STREAMEDは「業務委託料」。
> 三者全て異なる。CSV出力時は顧問先のMF登録名に合わせる必要あり。

### 差異3: 保険料 vs 損害保険料

| ソース | 科目名 |
|--------|--------|
| STREAMED | 保険料 |
| MF CSV（個人） | 損害保険料 |
| 本システム旧enum | INSURANCE（保険料） |

> MF個人は「損害保険料」、STREAMEDは「保険料」。
> MF法人は「保険料」。法人/個人で名称が異なる。

### 差異4: 貸倒金 vs 貸倒金(損失)

| ソース | 科目名 |
|--------|--------|
| STREAMED | 貸倒金 |
| MF CSV（個人） | 貸倒金(損失) |

> 半角括弧付きがMF正式名称。

### 差異5: 支払利息 vs 利子割引料

| ソース | 科目名 |
|--------|--------|
| STREAMED | 支払利息 |
| MF CSV（個人） | 利子割引料 |

> 完全に別名称。法人向けなら「支払利息」で合致する可能性あり。

## 5. STREAMED「科目コード」列のデフォルト税区分マッピング

STREAMEDの「科目コード」列は、本システムの`Account.defaultTaxCategoryId`に相当する。

| 科目コード値 | 本システムの概念ID | 対象科目例 |
|-------------|------------------|-----------|
| 自動判定(仕入) | `PURCHASE_TAXABLE_10`（AI判定） | 通信費、旅費交通費、消耗品費等 |
| 自動判定(売上) | `SALES_TAXABLE_10`（AI判定） | 売上高、雑収入 |
| 自動判定(売上貸倒) | `SALES_BAD_DEBT_10` | 貸倒金 |
| 非課税仕入 | `PURCHASE_NON_TAXABLE` | 法定福利費、保険料、租税公課、減価償却費、支払利息 |
| 非課税売上 | `SALES_NON_TAXABLE` | 受取利息 |
| 対象外 | `COMMON_EXEMPT` | 現金、普通預金、未払金、事業主貸等 |

> 「自動判定」= AIが8%/10%を判定する。デフォルトは10%。
> これはまさに `Account.defaultTaxCategoryId` + `Account.aiSelectable` の設計と一致。

## 6. 結論と設計判断

### 判断: 科目マスターは「MF正式名称」を正とする

```
STREAMED科目名  ≠  MF科目名  の場合がある（差異1-5）
→ CSV出力時にMFに送るのは「MF正式名称」
→ STREAMEDの科目名はUI表示用の参考
→ 本システムは概念IDで管理し、出力時にMF名称に変換
```

### 判断: 科目数は46が上限

```
MF全量: 120〜130科目
STREAMEDが使う: 46科目
本システム旧enum: 30科目

→ STREAMEDの46科目を基準にする
→ MF全量は不要（棚卸資産、繰延資産等はSTREAMEDの処理対象外）
→ 本システムの30→46への拡張で十分
```

### 判断: 法人CSVが取得不可能

```
法人向けMFアカウントがない → 正式名称を実機確認できない
→ 個人CSVで確認できた科目（34件一致）は信頼できる
→ 法人固有の科目名（短期借入金、未払費用、諸会費等）はSTREAMED表示名を暫定使用
→ 将来法人テストアカウント取得時に実機確認
```
