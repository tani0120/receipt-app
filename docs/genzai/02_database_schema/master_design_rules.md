## 確定ルール（13項目）

### ルール1: 適用期間

- 勘定科目・税区分ともに`effective_from`（適用開始日）と`effective_to`（適用終了日）を持つ
- 現役 = `effective_to`（適用終了日）が**空白（NULL）**
- 空白 = マスタとして有効

### ルール2: 権限制御

- マスタ管理はスタッフ管理画面で設定する**マスタ管理権限**で制御
- 権限がないスタッフは`/master`にアクセス不可
- スタッフ管理画面は**後で実装**（決定済み。[10_nullable_on_document_plan.md](file:///c:/dev/receipt-app/docs/genzai/10_nullable_on_document_plan.md) 優先度11 K6-K9）

#### 全管理ページ構成（2026-03-06 確定）

| # | 管理ページ | URL（想定） | 説明 |
|---|----------|------------|------|
| 1 | 顧問先管理 | `/admin/clients` | 顧問先の追加・編集・停止 |
| 2 | スタッフ管理 | `/admin/staff` | スタッフの追加・編集・停止。**管理権限の設定** |
| 3 | マスタ管理 | `/master` | ✅実装済み。勘定科目マスタ・税区分マスタ |
| 4 | 想定費用 | `/admin/billing` | 顧問先ごとの報酬設定 |
| 5 | 設定管理 | `/settings` | ✅実装済み。顧問先単位の勘定科目・税区分設定 |

#### 権限種別（2026-03-06 確定）

| 権限名 | 制御対象 | 説明 |
|--------|---------|------|
| **マスタ管理権限** | `/master` 配下全ページ | 勘定科目マスタ・税区分マスタの追加・修正・削除。**権限なし → アクセス不可** |
| **顧問先管理権限** | `/admin/clients` | 顧問先の追加・編集・停止 |
| **スタッフ管理権限** | `/admin/staff` | スタッフの追加・編集・停止・権限変更。**管理者のみ** |
| **費用管理権限** | `/admin/billing` | 報酬設定の閲覧・編集 |
| **設定管理権限** | `/settings` | 顧問先単位の勘定科目・税区分の表示/非表示・名称変更 |

> **最小権限**: 新規スタッフはデフォルトで全権限なし。管理者が個別に付与する。
> **管理者**: 全権限を持つ特別なロール。スタッフ管理権限を持つのは管理者のみ。

### ルール3: 論理削除とグレーアウト表示

- **物理削除禁止**。`deprecated`（非推奨フラグ）= `true` で無効化
- 勘定科目にも`deprecated`を追加（税区分と同様）
- 非推奨にした科目/税区分はマスタUI上で**グレーアウト表示**（選択不可だが見える）

### ルール4: 不変原則とデフォルト/カスタム科目の区分

- 税区分: 既存IDの税率・種類の変更は**禁止**
- 勘定科目・税区分はシステム提供の**デフォルト科目**とユーザーが追加した**カスタム科目**に分かれる

#### デフォルト科目 vs カスタム科目の編集権限（2026-03-06 確定）

| 操作 | デフォルト科目 | カスタム科目 |
|------|:---:|:---:|
| 名前編集 | ❌ 不可 | ✅ 可能 |
| 税区分変更 | ❌ 不可 | ✅ 可能 |
| 非推奨化 | ✅ 可能 | ✅ 可能 |
| 削除（物理） | ❌ 不可 | ✅ 可能 |
| 並び替え | ✅ 可能 | ✅ 可能 |

> **デフォルト科目**: `account-master.ts`/`tax-category-master.ts`に定義されたシステム提供科目（`isCustom = false`）
> **カスタム科目**: ユーザーが「追加」「コピー」で作成した科目（`isCustom = true`）
> カスタム科目のみ名前・税区分の編集と物理削除が可能

### ルール5: 名称変更時のMF警告

- **勘定科目**: 「この科目名を変更すると、MFインポート時に新しい勘定科目の登録または既存科目への変換を求められる可能性があります。」
- **税区分**: 「この名称を変更すると、MFインポート時に税区分警告が表示される可能性があります。対応はMFの指示に従ってください。」
- いずれも`?`マーク（ヘルプアイコン）で表示

### ルール6: 操作方式

- **行内編集（インライン）**: カスタム科目のみ、セルを直接クリックして編集可能
- デフォルト科目はインライン編集不可（セルクリックしても反応しない）
- **保存ボタン**で確定（即時保存ではない）
- **チェックボックス（✓）選択** → 削除・コピー・追加・復元が可能
  - コピー: チェック行の直下に「○○（コピー）」としてカスタム科目を挿入
  - 追加: チェック行の直下に「新規科目」（税区分: 対象外）としてカスタム科目を挿入
  - 削除: デフォルト科目は非推奨化のみ、カスタム科目は物理削除可能
  - 復元: 非推奨行を現役に戻す
- 実装時にユーザーに項目ごとに確認を出す

### ルール7: 顧問先への自動コピー（連動）

- 新規顧問先作成時、マスタの勘定科目・税区分を**自動コピー**
- 税区分は`default_visible`（デフォルト表示）= 有効の27件を自動挿入
- 勘定科目もマスタから自動コピー

### ルール8: マスタ変更の顧問先への伝播

| 変更種別 | 伝播方式 |
|---------|---------|
| **新規追加** | 自動反映 |
| **変更・非推奨** | 手動反映。通知＋スタッフ確認 |

### ルール9: AI可否の変更権限

- マスタの`aiSelectable`（AI自動選択可否）は**初期値のみ**設定
- 顧問先単位で**上書き可能**

### ルール10: ソート順の永続化

- ドラッグ＆ドロップで並び替え → `sort_order`（表示順）を更新 → DB保存
- 表示は常に `ORDER BY sort_order ASC`
- 新規科目追加時は一番下に追加（`sort_order` = 最大値 + 1）
- 将来の更新API: `PATCH /accounts/sort` で`id`と`sort_order`の配列を送信

### ルール11: デフォルト税区分変更の顧問先連動

- ルール8に準じる（変更は手動通知＋スタッフ反映）

### ルール12: 税制改正時の強制適用

- **税区分の変更（=税制改正）のみ**全顧問先に**強制適用**
- **勘定科目は全て手動反映**（強制適用の対象外）
- 管理者が「一括切替」を実行 → 確認画面 → 全顧問先に適用
- 通常の科目追加・名称変更とは**別ルート**

#### 強制/手動の区分基準: 税区分の変更 = 税制改正のみ

| 変更種別 | 対象 | 強制/手動 | 理由 |
|---------|------|----------|------|
| 税率変更（法改正） | 税区分 | ✅ 強制 | 法律改正。全事業者に適用義務 |
| 税区分の非推奨化（法改正に伴う） | 税区分 | ✅ 強制 | 税率変更とセット |
| 税区分の新規追加（税制改正に伴う） | 税区分 | ✅ 強制 | 法改正に伴う新税区分 |
| 勘定科目の新規追加 | 勘定科目 | 📩 手動 | 事務所が使う/使わないを判断 |
| 勘定科目の名称変更 | 勘定科目 | 📩 手動 | MF整合性に影響 |
| 勘定科目の非推奨化 | 勘定科目 | 📩 手動 | 既存仕訳に影響 |
| AI可否の変更 | 両方 | 📩 手動 | 顧問先上書き可能のため参考情報 |

### ルール13: マスタ変更履歴

- **別テーブル**（`master_change_log`）で履歴を管理
- 元テーブルを汚さない
- 変更前後の値（before / after）をJSON保存

#### テーブル構造

| カラム | 内容 |
|--------|------|
| `id` | 主キー |
| `table_name` | 変更対象テーブル名 |
| `record_id` | 変更されたレコードID |
| `action` | `INSERT` / `UPDATE` / `DEPRECATE` |
| `before_data` | 変更前JSON |
| `after_data` | 変更後JSON |
| `modified_by` | 変更者 |
| `modified_at` | 変更日時 |

## 補助科目の扱い（確定）

- マスタ（`/master/accounts`）: 補助科目列は**常に空白**。ヘルプアイコンで「顧問先ごとの設定で入力します」と表示

- UI上の「分類」列は**廃止**（2026-03-06実施済み）
- データ層の`category`フィールドはフィルタリング用に保持（UIには非表示）
