# テスト自動化計画

**作成日**: 2026-02-27
**目的**: 手動ブラウザ確認のみのUI検証を自動テスト化
**解決時期**: Phase B（構造固定）
**移管元**: `04_mock/task_phase_a.md` §5.1 E1-E3（2026-02-27移管）

---

## E1: ソート全列動作テスト

- **現状**: 手動ブラウザ確認のみ
- **対象**: 全24列のソート（昇順・降順）
- **検証内容**:
  - 各列クリックで正しいソート順になること
  - ソート方向トグル（昇順⇔降順）が動作すること
  - amount列の数値ソート、text列の文字列ソートが正しいこと
  - component列（学習、証票、警告等）の重み付けソートが正しいこと

---

## E2: モーダル全種動作テスト

- **現状**: 手動ブラウザ確認のみ
- **対象**: 画像プレビューモーダル、過去仕訳検索モーダル、コメントモーダル
- **検証内容**:
  - 各モーダルの開閉が正常に動作すること
  - ドラッグ移動が動作すること
  - モーダル内の操作（検索、フィルタ、入力）が正常に動作すること
  - 画面外clampが動作すること

---

## E3: トグル動作テスト

- **現状**: 手動ブラウザ確認のみ
- **対象**: 未読/既読トグル、対象/対象外トグル、ゴミ箱/復活、コピー
- **検証内容**:
  - 個別操作（⋯ドロップダウン）が正常に動作すること
  - 一括操作（L4バーアクションモード）が正常に動作すること
  - exported行の制限（disabled + グレーアウト）が正常に動作すること
  - フィルタとの連動（表示/非表示切替）が正常に動作すること
