## 判定基準

| 評価軸 | 点数定義 |
|--------|----------|
| **実装有無（Evidence）** | 0=未実装, 1=死骸・部分実装(1レイヤー), 3=複数レイヤー存在, 5=フル実装+テスト |
| **インパクト** | 0=なし, 1=1ファイル, 2=2-5ファイル, 3=1コンポーネント, 4=複数コンポーネント, 5=全体影響 |
| **非機能要件** | 0=問題なし, 1=軽微, 2=中程度, 3=パフォーマンス, 4=セキュリティ, 5=コスト大 |
| **業務価値** | 0=なし, 1=便利, 2=効率化, 3=売上貢献, 4=顧客満足, 5=事業必須 |

## スコアリング
各項目 0‑5 点、合計 0‑20 点
- **15 点以上** → **昇格**（Judgment UseCase に追加）
- **10‑14 点** → **保留**（追加情報取得後再評価）
- **9 点以下** → **除外**（次回見直しまで保持）

## 使い方

1. このテーブルを **スプレッドシートにコピー＆ペースト**
2. **インパクト / 非機能要件 / 業務価値** の空欄に **0-5 の点数** を入力
3. 合計点に基づき **判定列** に `昇格 / 保留 / 除外` を記入
4. 完了したらチャットに貼り付けて AI に送信
