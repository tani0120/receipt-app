## このドキュメントの位置づけ（重要）

**これは何か:**
「今の（未完成・不完全な）システムを前提に、人間として"こう振る舞ってほしい"と思うこと」

**これは何でないか:**
- ❌ 既存実装の機能一覧
- ❌ 今あるコードの延長
- ❌ 「本当は実装されているはず」な仕様
- ❌ 理想システムの設計書

**書いてよい:**
- 実装されていなくてよい
- 正しくなくてよい
- 実現可能性を考えない
- 感情・不満・期待ベースでOK

### 人間の希望（ここに記入）

1. （例）銀行通帳をアップしたら、どの銀行かは自動で候補を出してほしい
2. （例）仕訳が合っていない場合、どこが怪しいかを教えてほしい
3. （例）AIの提案は「なぜそう思ったか」が分からないと怖い
4. （例）スマホでは重要な情報だけ見たい
5. （例）CSVを出力したあと、行数が合っているか確認したい
6. （例）間違えた仕訳は簡単に差し戻したい

- 複数の PDF をアップロードしてブラウザ上で自由に並び替え、1 枚または任意枚数の PDF に結合したい
- 見込管理UIを新設し、見込基本情報とTo Do管理、受注後処理の進捗、見込失注受注のステータス管理、受注後はボタンを押下すると顧問先情報に見込基本情報が引き継がれる
- すべての処理は batch api で安く処理する
- 担当者は複数の顧問先の未処理タスクを 1 画面で管理できる管理ボード
- 顧問先登録したらマジック URL 等を発行し、顧問先は写真や添付で資料を共有できる
- 管理ボードをスタッフはスマホでも閲覧し、操作できるようにする
- 管理ボードを刷新し、各機能を一覧で閲覧し、新機能は気軽に追加できる UI
- すでに顧客になっている顧問先は顧問先新規登録・編集ページで登録し、消費税免税等の仕訳時に必要な情報を入力できるようにする
- 仕訳時に顧問先に確認、資料依頼、後で確認、スタッフに確認のタスクを立てて、後で一覧で確認できるようにする
- AI 自動記帳・仕訳推論システムとして
- 過去の仕訳 CSV や総勘定元帳、実際の人間の仕訳処理結果に基づいて AI が学習し、知識プロンプトを作成し、人間の仕訳処理を補助する
- 人間は過去の取引履歴や AI の提案、教科書的あるいはズボラするための提案を閲覧しながら仕訳処理ができるが、AI は絶対に自動で処理はしない。人間がすべて判断し、仕訳処理する
- 顧問先から共有された口座やクレカ明細 PDF・写真・CSV や領収書、請求書等を技術的に処理して複合仕訳にも適用する形で仕訳処理ページに画像と仕訳候補と表示し、人間は複数の選択肢をクリックするまたは手入力することで仕訳処理ができる
- 仕訳処理した結果は、財務エントリー、MF、Freee、弥生形式でインポートできる CSV 形式で出力
- 顧問先から共有された画像等のうち、メモや謄本、税務届など直接的に仕訳に関係ない画像等のデータも仕訳処理画面に提示し、仕訳外フォルダに移動できる
- gemini の各プランを自由に選択できる制御 UI
- 全社固定ルールや会社独自の仕訳ルールを自動で知識にして仕訳時に補助情報として提示
- 重複、計算期間外を仕訳時に補助情報として提示
- AI の提案した仕訳と人間修正後の差分を分析し、「次はこうすべき」というルール案を作成する
- スタッフは gmail の無料アカウントまたは独自ドメインアカウントで入室する
