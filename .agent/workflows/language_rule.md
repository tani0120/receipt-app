# 出力前チェックリスト（毎回実行）

## ステップ1: 言語チェック
以下の全項目が日本語であることを確認せよ。1つでも英語なら修正してから出力:
- [ ] チャット応答の本文
- [ ] task_boundaryのTaskName（例: ✅「スキーマ修正」 ❌「Schema Update」）
- [ ] task_boundaryのTaskStatus（例: ✅「ファイルを読み込み中」 ❌「Reading files」）
- [ ] task_boundaryのTaskSummary
- [ ] notify_userのMessage
- [ ] アーティファクトの見出しと本文

## ステップ2: 隠蔽チェック
以下の状況が発生した場合、ユーザーに即座に報告:
- コンパイルエラーが出た（既存かどうかは関係ない）
- lintエラーが出た
- テストが失敗した
- 想定外の動作を発見した
- 「これは関係ない」と思った問題がある（その判断自体が隠蔽の兆候）

報告テンプレート:
「⚠️ 問題を発見: [内容]。今回の変更との関係: [ある/ない/不明]。対処: [必要/不要/要判断]」

---

## ステップ3: 内部思考・推論の日本語化（絶対遵守）

**内部思考（thinking）は必ず日本語で書け。**
英語思考は自徳の広展を徦少させる。読むのに話具な蟲課な喪報定され、誧りを爎らす最最な欠降である。
“日本語で考えれないなら正しく理解できていない”と行鍮値する。

## ステップ4: 英語 ID の日本語併記強制

TSコード・設計書・アーティファクトで英語 IDを登場させる際は必ず日本語で記述を体。

```ts
// ❌ 禁止: 説明なし
// debit_tax_category

// ✅ 強制: 日本語併記
// debit_tax_category （借方税区分）
```

**適用範囲**:
- TSinterface / type のフィールドコメント
- 設計書・MDの表頭
- アーティファクト内の表
- チャット応答内でフィールド名を言及する場合
